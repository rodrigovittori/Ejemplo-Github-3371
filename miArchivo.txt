NOTA: Normalmente la versión principal se conoce como "main" o "master"

Algunas ramas comunes son:

> dev -> "desarrollo" > donde se desarrollan nuevas características
> test -> "pruebas" > donde se testean esas características

> stable/alpha/beta > versiones más estables o experimentales que se publican
                       para tener Long-Term-Support (LTS)
                       o bien obtener datos de distintos entornos de usuario
